## 2024-05-18 - Icon-Only Button Accessibility
**Learning:** Found several icon-only interactive buttons in `TopBar.jsx` and `UnifiedFeed.jsx` that were lacking both `aria-label` definitions (making them invisible/unclear to screen readers) and `focus-visible` states (making them difficult to track for keyboard navigators). Some buttons were only visible on `group-hover`, making them completely inaccessible via keyboard.
**Action:** When adding or encountering icon-only interactive elements, ensure they have descriptive `aria-label` attributes and implement clear `focus-visible` styling (e.g., `focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-primary`). If an element is hidden by default and shown on hover, ensure it is also visible on focus (e.g., `focus-visible:opacity-100`).
